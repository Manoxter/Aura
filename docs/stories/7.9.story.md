# Story 7.9 — Rate limiting /api/ai/*: 60 req/hora por tenant_id
**Épico:** EP-07 IA Klauss e Rotas de Decisão
**Sprint:** F-CICD
**Status:** Done
**Agentes:** @security-auditor, @devops (Gage), @dev (Dex)

---

## User Story
Como operador do Aura,
quero que todos os endpoints `/api/ai/*` tenham rate limiting de 60 requisições/hora por tenant,
para que nenhum usuário esgote a cota da API Groq e prejudique outros tenants.

---

## Acceptance Criteria
- [x] AC-1: Todos os endpoints `/api/ai/*` verificam rate limit antes de chamar a IA
- [x] AC-2: Limite: 60 req/hora por `user.id` (tenant_id)
- [x] AC-3: Resposta 429 com body `{ error, retryAfter, limit, window }` ao exceder
- [x] AC-4: `checkRateLimit(userId, route, limit, windowMs)` implementado em `src/lib/api/rate-limit.ts`
- [x] AC-5: TypeCheck: 0 erros
