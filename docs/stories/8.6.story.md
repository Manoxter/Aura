# Story 8.6 — Validação Zod em todos os bodies POST/PUT das API routes
**Épico:** EP-08 Infra/DB
**Sprint:** F-CICD
**Status:** Done
**Agentes:** @security-auditor, @dev
**Pré-requisito:** SaaS-6 Done (Zod já instalado)

---

## User Story
Como engenheiro de segurança,
quero que todas as API routes que recebem dados via POST/PUT tenham schemas Zod de validação,
para que inputs malformados ou maliciosos sejam rejeitados com mensagens descritivas antes de chegar à camada de banco de dados.

---

## Acceptance Criteria
- [x] AC-1: Schema Zod criado para endpoints ativos: `/api/invite` (POST) + `/api/[projetoId]/riscos` (POST). Todos os endpoints AI já tinham Zod. Endpoints `/api/projects`, `/api/tarefas`, `/api/orcamento`, `/api/progresso` não existem neste projeto.
- [x] AC-2: Requisição com body inválido retorna HTTP 400 com mensagem descritiva dos campos inválidos (via validateBody helper).
- [x] AC-3: 0 (zero) ocorrências de `any` nos tipos de request body nos endpoints modificados.
- [x] AC-4: TypeCheck limpo — 0 erros novos introduzidos.
