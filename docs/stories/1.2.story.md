## User Story
Como motor matemático do Aura,
quero calcular o NVO (Núcleo Viável Ótimo) usando uma hierarquia de 3 níveis dependendo da classificação do triângulo,
para que o MATED sempre meça uma distância com significado geométrico e gerencial.

## Acceptance Criteria
- [x] AC-1: TA acutângulo → NVO = baricentro do triângulo órtico de TA
- [x] AC-2: TA obtusângulo → NVO = baricentro do TM (ponto fixo do plano original)
- [x] AC-3: TM também obtusângulo (caso patológico) → NVO = incentro do TM
- [x] AC-4: Função `classificarTriangulo(lados)` retorna `'acutangulo' | 'obtusangulo' | 'retangulo'`
- [x] AC-5: MATED calculado corretamente para os 3 casos
- [x] AC-6: Suite Big Dig: resultado de MATED igual ou melhor que versão anterior
- [x] AC-7: Novo teste unitário para triângulo obtusângulo-fixture

## Riscos
| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| Triângulo órtico só existe para triângulos acutângulos — implementação incorreta da detecção pode calcular baricentro de figura inválida | Alta | Alto | Implementar guard: se não acutângulo, não calcular ortocentro; testar com fixture obtusângulo primeiro |
| MATED depende do NVO — mudança na hierarquia pode alterar resultados da suite Big Dig | Média | Alto | Executar Big Dig com modelo A (NVO antigo) e B (novo) antes de substituir; comparar delta |
