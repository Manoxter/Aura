# Story 5.5 — Alerta Automático: TA/TM Desvia > 5% (Limiar Configurável)

## User Story
Como PM usando o Aura,
quero receber um alerta automático quando o desvio entre TA e TM supera 5% (limiar configurável),
para que problemas sejam detectados antes de atingir a zona CRISE.

## Acceptance Criteria
- [x] AC-1: Após recalcular TA, verificar se MATED > limiar configurado (default: 5% = 0.05)
- [x] AC-2: Limiar configurável por projeto nas configurações do TAP
- [x] AC-3: Alerta tipado: `{ tipo: 'desvio_clinico', mated: number, zona: 'SEGURO'|'RISCO'|'CRISE', limiar: number }`
- [x] AC-4: Toast na UI com ação sugerida: "Ver MATED" ou "Consultar Klauss"
- [x] AC-5: Alerta persistido em `decisoes_mated` com `impacto_area_percent`
- [x] AC-6: Klauss notificado (via contexto) para sugerir ação quando alerta for gerado
- [x] AC-7: Alerta não repete se MATED permanece na mesma faixa (apenas nova entrada de zona gera alerta)
- [x] AC-8: PM pode silenciar alerta por 24h (badge amarelo na sidebar)

## Riscos
| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| Throttle por zona usando `decisoes_mated` requer query a cada recalcular TA — se projeto tem muitos registros, latência pode impactar UX | Média | Médio | Memoizar última zona detectada em memória (React state/context) para comparar sem query; persistir em DB apenas quando zona muda |
| `limiar_alerta` por projeto exige campo novo no schema de projetos ou no TAP — sem migration, campo não existe e código falha | Alta | Alto | Verificar se campo já existe antes de implementar; criar migration mínima se necessário, ou usar default 0.05 hardcoded com fallback |
