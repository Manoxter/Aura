## Contexto
Integrar a Base de Dados (Supabase) e os logs de Performance/Matemáticos nas Esteiras de Integração Continua (CI/CD) para automatizar testes e garantir integridade na nova arquitetura geométrica escalena que está substituindo o equilátero mandatório.

## Objetivos (Acceptance Criteria)
- [ ] Atualizar o trigger de "criarVersaoInicial" para gravar via RPC ou DB-Call os JSONs expandidos das "Áreas Integradas de Prazo e Custo" criados em `math.ts`. 
- [ ] A tabela no DB do TMP (Matriz Versionado) deve prever um campo Boolean ou Enum que classifique se aquele Baseline nasceu como Obtusângulo, Acutângulo ou Retângulo (para monitoramento Klauss).
- [ ] Criar workflow de Git Actions (`.github/workflows/main.yml`) integrando o linter + vitest + push para branch Dev/Preview antes de soltar pra Produção Vercel. 
- [ ] Estabelecer protocolos contínuos para rate control com endpoints serverless interagindo com Supabase em projetos carregados na nuvem.
