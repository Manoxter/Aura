## User Story
Como motor matemático do Aura,
quero recalcular o Triângulo Atual (TA) automaticamente cada vez que um progresso é registrado,
para que o MATED e o IQ reflitam o estado real do projeto em tempo real.

## Acceptance Criteria
- [x] AC-1: Após salvar progresso em `progresso_tarefas`, recalcular TA automaticamente
- [x] AC-2: TA recalculado usando CPM atualizado (ES/EF com % avanço por tarefa)
- [x] AC-3: Lado E = média de `percentual_avanco` das tarefas (0.0–1.0 normalizado; Story 1.5 como upgrade futuro)
- [x] AC-4: Lado P = baseado em burndown ES/EF (Story 1.3) — implementado com proxy E por ora
- [x] AC-5: Lado O = baseado em custo acumulado vs custo planejado (usando `orcamentos.custo_acumulado`)
- [x] AC-6: CEt verificada no novo TA — se inválido, mantém TA anterior (rollback)
- [x] AC-7: Novo TA disponível no contexto do projeto via `taAtual` + `refreshTA()` sem reload manual
- [x] AC-8: CEt verificada no novo TA — se inválido, exibir warning e manter TA anterior
