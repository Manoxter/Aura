# Story 13.3 — API Routes: POST /riscos + DELETE /riscos/:id
**Épico:** EP-13 Prometeu Extrínseco
**Sprint:** PROMETEU
**Status:** Done
**Agentes:** @dev (Dex), @security-auditor

---

## User Story
Como PM do projeto,
quero que as operações de criação e exclusão de riscos sejam feitas via API route segura,
para que a lógica de persistência seja centralizada e protegida por autenticação.

---

## Acceptance Criteria
- [x] AC-1: `POST /api/[projetoId]/riscos` aceita `{ titulo, categoria, probabilidade, impacto }` e insere com `score_rc` calculado.
- [x] AC-2: `DELETE /api/[projetoId]/riscos/[riscoId]` remove o risco via RLS (tenant_id = auth.uid()).
- [x] AC-3: Ambas as rotas retornam 401 se o usuário não estiver autenticado (via `requireAuth`).
- [x] AC-4: POST valida categoria (5 valores), probabilidade e impacto em [0, 1]; retorna 422 se inválido.
- [x] AC-5: TypeCheck limpo — 0 erros novos introduzidos.
