## User Story
Como PM usando o Aura,
quero ver na UI quantos projetos embasam os limiares de risco apresentados,
para que eu confie na régua MATED e entenda se estou vendo dados genéricos ou calibrados para meu setor.

## Acceptance Criteria
- [x] AC-1: Badge discreto no painel MATED: "Régua calibrada com N projetos do setor [Setor]"
- [x] AC-2: Tooltip no badge: detalha "Literatura (PMI/Flyvbjerg/World Bank)" quando N < 30
- [x] AC-3: Tooltip quando N ≥ 30: "Calibração empírica: baseado em projetos reais do seu tenant"
- [x] AC-4: Badge usa cor neutra (não confundir com alerta)
- [x] AC-5: Badge visível nas páginas: `/motor/cdt/` (Triângulo Matriz) e `/decisao/mated/`
- [x] AC-6: N atualiza após arquivamento (sem reload manual necessário)
- [x] AC-7: PM não vê σ nem valores técnicos internos — apenas "N projetos" e fonte

## Tasks

## Riscos
| Risco | Probabilidade | Impacto | Mitigação |

| `getModeInfo()` (Story 3.4) pode não estar disponível como hook reativo — badge pode exibir N desatualizado após arquivamento | Média | Médio | Chamar `getModeInfo()` com invalidação de cache no evento de arquivamento; ou usar Supabase realtime subscription |
