## User Story
Como sistema de calibração do Aura,
quero priors estatísticos de MATED por setor embasados em literatura publicada (PMI, Flyvbjerg, World Bank),
para que as zonas de risco sejam calibradas com base em ≥1000 projetos históricos desde o primeiro uso.

## Background
- **PMI Pulse of the Profession**: dados sobre cost overrun e schedule slip por setor
- **Bent Flyvbjerg**: estatísticas de megaprojetos (construção, TI, infraestrutura)
- **World Bank**: projetos de infraestrutura em países em desenvolvimento
- @roberta é responsável pela metodologia e rastreabilidade bibliográfica

## Acceptance Criteria
- [x] AC-1: 4 setores mapeados: `construcao_civil`, `tecnologia`, `infraestrutura`, `saude`
- [x] AC-2: Cada setor tem valores `mated_medio`, `sdo_score`, `sigma_setor` derivados da literatura
- [x] AC-3: Cada prior tem `fonte` rastreável (citação bibliográfica no campo `descricao`)
- [x] AC-4: @roberta assina metodologia (critério de inclusão, cálculo dos valores)
- [x] AC-5: Script de seed SQL com INSERT documentado por fonte
- [x] AC-6: `SELECT setor, mated_medio, fonte FROM aura_calibration_events WHERE tipo='prior'` retorna 12+ linhas
- [x] AC-7: `sigma_setor` (coluna nova se necessário) armazenado por setor para Story 3.4
