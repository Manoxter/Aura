# Story 7.5 — Klauss injeta histórico de decisões MATED no contexto (entre sessões)
**Épico:** EP-07 IA Klauss e Rotas de Decisão
**Sprint:** KLAUSS-MATED
**Status:** Done
**Agentes:** @dev (Dex), @aura-klauss, @jordy

---

## User Story
Como PM usando o Klauss no Aura,
quero que o Klauss conheça as últimas decisões MATED do meu projeto ao iniciar uma nova conversa,
para que eu não precise repetir o contexto de cada sessão.

---

## Acceptance Criteria
- [x] AC-1: Endpoint `/api/ai/klauss` aceita `projetoId` (UUID opcional) no body
- [x] AC-2: Se `projetoId` fornecido, busca as 5 últimas decisões de `decisoes_mated` no Supabase
- [x] AC-3: Decisões injetadas no system prompt como `[HISTORICO_DECISOES]`
- [x] AC-4: Formato: `N. [zona_resultado] Impacto XX.X% — "descricao (max 120 chars)"`
- [x] AC-5: Se nenhuma decisão encontrada, seção não aparece no prompt
- [x] AC-6: `AiKlaussRequestSchema` inclui `projetoId` como UUID opcional
- [x] AC-7: TypeCheck: 0 erros
