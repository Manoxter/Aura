## User Story
Como desenvolvedor contribuindo para o Aura,
quero uma documentação visual dos componentes do Design System com exemplos interativos,
para que eu possa ver como cada componente se parece e como usá-lo sem precisar rodar o app.

## Background
Após DS-1 a DS-9, o Aura terá um Design System funcional mas sem documentação visual. Um desenvolvedor novo ainda precisaria abrir o código para entender como usar `Toast`, `EmptyState`, `SetupStepper`, etc.

A decisão é criar uma página dedicada `/design-system` no próprio app Next.js (não Storybook standalone) que serve como:
1. Referência visual de todos os componentes
2. Prova de vida que os tokens estão corretos
3. Playground para desenvolvedores

Storybook foi considerado mas rejeitado por ser overhead demais para a fase atual. A página interna é suficiente e roda no mesmo servidor.
