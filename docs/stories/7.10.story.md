# Story 4.0 — Advisor de Compressão IA: Fast-Tracking e Crashing via CPM

**Épico:** EP-04 — Inteligência Adaptativa de Cronograma
**Status:** Draft
**Criado por:** Orion (@aiox-master)
**Data de criação:** 2026-03-26
**Prioridade:** ALTA — habilita tomada de decisão proativa de compressão de prazo

---

## Contexto

O Aura já possui CPM calculado com ES/EF/LS/LF/Folga por tarefa e o triângulo CDT como monitor de saúde do projeto. No entanto, quando o CPM calculado supera o baseline, o operador não tem orientação automatizada sobre **quais ações tomar** e **qual o custo/risco de cada uma**.

Este módulo implementa um botão IA no painel Tarefas e Diagramas que, ao ser acionado, lê a rede CPM e produz um relatório de compressão com duas dimensões:

1. **Fast-Tracking**: paralelização de tarefas usando folgas existentes (custo zero, risco de retrabalho)
2. **Crashing**: redução de duração em tarefas críticas via adição de recursos (custo direto, impacto CDT)

---

## User Story

**Como** gerente de projeto usando o Aura,
**Quero** acionar um botão IA que analise o cronograma CPM e sugira ações de compressão,
**Para que** eu saiba exatamente em quais tarefas posso aplicar fast-tracking ou crashing, qual o custo estimado e como isso impacta o triângulo CDT antes de decidir.

---

## Acceptance Criteria

### AC-1 — Botão de Acionamento
- [ ] Botão "Analisar Compressão" visível no painel Tarefas e Diagramas, ao lado do botão "Calcular CPM"
- [ ] Botão desabilitado quando CPM não foi calculado (tarefas sem ES/EF preenchidos)
- [ ] Tooltip explica pré-requisitos quando desabilitado

### AC-2 — Análise de Fast-Tracking
- [ ] Identifica pares de tarefas com relação FS (Finish-Start) onde a segunda tem folga livre ≥ N dias
- [ ] Sugere conversão para SS (Start-Start) com offset = duracao_predecessor - folga_disponivel
- [ ] Para cada sugestão mostra: Tarefa A → Tarefa B, dias ganhos, folga consumida, risco (BAIXO/MÉDIO/ALTO)
- [ ] Risco ALTO quando tarefa está no caminho crítico após a paralelização ou folga < 10% da duração

### AC-3 — Análise de Crashing
- [ ] Lista todas as tarefas do caminho crítico ordenadas por menor custo/dia de redução
- [ ] Calcula custo de crashing como: custo_tarefa × fator_crashing (default 15-25% por dia comprimido, configurável por tipo de projeto)
- [ ] Mostra impacto acumulado: total de dias ganhos × custo total de crashing
- [ ] Alerta quando crashing em uma tarefa desloca o caminho crítico para outra sequência

### AC-4 — Preview de Impacto no CDT
- [ ] Modal exibe mini-triângulo CDT mostrando posição atual (TA) e posição projetada após compressão
- [ ] Alerta visual quando compressão proposta empurra TA para zona RISCO ou CRISE (IQ < 80%)
- [ ] Alerta quando ε (epsilon) se aproxima de 90° (Estado Singular da Síntese de Clairaut)

### AC-5 — Relatório Exportável
- [ ] Relatório pode ser visualizado em modal lateral (side panel) com scroll
- [ ] Seções: Resumo Executivo | Fast-Track Opportunities | Crashing Analysis | CDT Impact Preview
- [ ] Botão "Exportar PDF" ou "Copiar como Texto" para o relatório

### AC-6 — Base de Conhecimento PMI/PMBOK
- [ ] Cada sugestão inclui referência à boa prática PMI aplicável (ex: "PMBOK 6a ed., Sec. 6.6.2.3 — Schedule Compression")
- [ ] Tipos de projeto configuráveis: Construção Civil, TI, Infraestrutura, Geral (afetam fatores de crashing e limiares de risco)

## Escopo

### IN
- Botão no painel Tarefas e Diagramas
- Modal lateral com relatório de compressão
- Análise de fast-tracking (baseada em folgas existentes do CPM)
- Análise de crashing (baseada em custo por tarefa e fatores PMBOK)
- Preview de impacto no triângulo CDT (cálculo client-side)
- Alertas de Síntese de Clairaut (Estado Singular, Obtuso β/γ)
- Base de fatores de crashing por tipo de projeto (local, não LLM)

### OUT (nesta story)
- Integração com LLM/AI externa para análise semântica das tarefas
- Edição automática do cronograma pelo advisor (somente sugestões, não edita)
- Análise de resource leveling (nivelamento de recursos)
- Histórico de compressões aplicadas

## Design Técnico Proposto

### Nova função engine
```typescript
// src/lib/engine/compressao.ts
export function analisarCompressao(
  tarefas: Tarefa[],
  prazoBase: number,
  custoBase: number,
  tipoProjetoFator: 'construcao' | 'ti' | 'infraestrutura' | 'geral'
): RelatorioCompressao

interface RelatorioCompressao {
  cpmAtual: number
  diasExcedentes: number
  fastTrack: FastTrackOp[]
  crashing: CrashingOp[]
  impactoCDT: ImpactoCDT
  alertasSC: string[]
}
```

- Arquivo: `src/app/(dashboard)/[projetoId]/setup/tarefas-diagramas/page.tsx`

## Riscos

| Risco | Probabilidade | Mitigação |
|-------|--------------|-----------|
| CPM não calculado antes de acionar advisor | Alta | Botão desabilitado por AC-1 |
| Fatores de crashing muito imprecisos sem dados históricos | Média | Usar ranges PMBOK com disclaimer |
| Sugestão de fast-track gerar conflito de predecessoras no UI | Média | Advisor é read-only, não edita automaticamente |
