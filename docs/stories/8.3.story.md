# Story 8.3 — Migration: Trigger `updated_at` Automático
**Épico:** EP-08 Infraestrutura
**Sprint:** DB-EXEC
**Status:** Done
**Agentes:** @data-engineer (Dara)
**Decisão:** DB13

## User Story
Como banco de dados do Aura,
quero um trigger automático que atualiza `updated_at` em todas as tabelas,
para que o timestamp de modificação seja sempre confiável sem dependência do frontend.

## Acceptance Criteria
- [x] AC-1: Função `set_updated_at()` criada no schema public
- [x] AC-2: Trigger `trg_set_updated_at` ativo em `projetos`
- [x] AC-3: Trigger ativo em `tarefas`
- [x] AC-4: Trigger ativo em `eap_nodes`
- [x] AC-5: Trigger ativo em `triangulo_matriz_versoes`
- [x] AC-6: Trigger ativo em `progresso_tarefas`
- [x] AC-7: Trigger ativo em `decisoes_mated`
- [x] AC-8: UPDATE em qualquer tabela acima incrementa `updated_at` automaticamente

## Tasks
- [ ] 1. Criar arquivo `supabase/migrations/YYYYMMDD_add_updated_at_trigger.sql`
- [ ] 2. Escrever função PL/pgSQL `set_updated_at()` retornando NEW com NEW.updated_at = NOW()
- [ ] 3. Criar trigger BEFORE UPDATE para cada tabela listada
- [ ] 4. Garantir que tabelas já têm coluna `updated_at TIMESTAMPTZ` — adicionar se ausente
- [ ] 5. Testar: UPDATE em projetos → confirmar `updated_at` alterado
- [ ] 6. Aplicar via `supabase db push`

## File List
- `supabase/migrations/YYYYMMDD_add_updated_at_trigger.sql`

## Definition of Done
- [ ] Função `set_updated_at()` criada
- [ ] Triggers ativos em todas as 6 tabelas
- [ ] Teste manual confirma incremento automático
- [ ] Migration listada em `supabase migration list`

## Escopo
**IN:** Função PL/pgSQL `set_updated_at()`, triggers BEFORE UPDATE em 6 tabelas (projetos, tarefas, eap_nodes, triangulo_matriz_versoes, progresso_tarefas, decisoes_mated), adição de coluna `updated_at TIMESTAMPTZ` onde ausente.
**OUT:** Não inclui trigger para `created_at` (assumido preenchido no INSERT), não cobre tabelas de calibração (`aura_calibration_events`, `aura_setor_config`) que não precisam de `updated_at`, não implementa auditoria de qual campo foi alterado.

## Riscos
| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| Tabelas `triangulo_matriz_versoes` e `progresso_tarefas` (Story 8.1) devem existir antes desta migration — se Story 8.1 não estiver Done, migration falhará | Alta | Médio | Confirmar Story 8.1 como Done; adicionar IF EXISTS antes de criar triggers |
| Se coluna `updated_at` já existe em alguma tabela com tipo diferente de TIMESTAMPTZ, ALTER TABLE falhará por tipo incompatível | Baixa | Médio | Verificar schema atual de cada tabela antes de executar; usar `ALTER COLUMN` com CAST se necessário |
