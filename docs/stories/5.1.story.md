# Story 5.1 — % Avanço por Tarefa: Input em Gerenciamento e Kanban

**Sprint:** EXEC-MODULE
**Status:** Done
**Agentes:** @dev (Dex), @ux-design-expert (Uma)

## User Story
Como PM usando o Aura,
quero registrar o percentual de avanço de cada tarefa diretamente na página de Gerenciamento e no Kanban,
para que o sistema tenha dados reais para recalcular o Triângulo Atual.

## Acceptance Criteria
- [x] AC-1: Input numérico (0–100%) em cada linha da tabela de Gerenciamento para % avanço
- [x] AC-2: Mesma funcionalidade disponível no card do Kanban (tooltip ou campo inline)
- [x] AC-3: Valor salvo em `progresso_tarefas` com `registrado_em` e `registrado_por` (auth.uid())
- [x] AC-4: RLS: somente o tenant do projeto pode salvar progresso
- [x] AC-5: Último valor registrado aparece pré-preenchido no input (não zero toda vez)
- [x] AC-6: Validação: valor fora de 0–100 bloqueado com mensagem de erro
- [x] AC-7: Toast de confirmação ao salvar: "Progresso da tarefa '[nome]' atualizado"
