# Story 7.8 — Structured output JSON schema para predecessoras
**Épico:** EP-07 IA Klauss e Rotas de Decisão
**Sprint:** KLAUSS-MATED
**Status:** Done
**Agentes:** @jordy, @dev (Dex)

---

## User Story
Como usuário do endpoint `/api/ai/predecessors`,
quero que a IA retorne predecessoras em formato JSON estruturado e validado,
para que o frontend possa processar as sugestões de forma confiável sem parsing frágil de texto livre.

---

## Acceptance Criteria
- [ ] AC-1: Endpoint `/api/ai/predecessors` retorna JSON estruturado com schema definido
- [ ] AC-2: Schema: `{ tasks: [{ id: string, predecessoras: string[] }] }`
- [ ] AC-3: Groq chamado com `response_format: { type: 'json_object' }` ou equivalente
- [ ] AC-4: Response validado com Zod antes de retornar ao cliente
- [ ] AC-5: Fallback gracioso se Groq não retornar JSON válido (re-tentativa ou erro 422)
- [ ] AC-6: TypeCheck: 0 erros

---
