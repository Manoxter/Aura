## User Story
Como motor Monte Carlo do Aura,
quero que σ (desvio padrão das simulações) seja parametrizado por setor usando dados da literatura,
e que o sistema migre automaticamente para σ empírico quando o tenant acumular ≥30 projetos arquivados.

## Background
- σ = 0.1 (global anterior) substituído por σ por setor com base em Flyvbjerg et al.
- Construção Civil: σ ≈ 0.20–0.25 (alta dispersão)
- Tecnologia: σ ≈ 0.15–0.18
- Infraestrutura: σ ≈ 0.18–0.22
- Saúde: σ ≈ 0.10–0.13
- Após n≥30: σ = `STDDEV(mated_medio)` dos projetos empíricos do tenant/setor
- @roberta tomou responsabilidade técnica sobre esta decisão (sessão anterior)

## Riscos
| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| `STDDEV()` no Supabase com n=30 projetos pode retornar NULL se todos os mated_medio forem idênticos — função deve tratar NULL com fallback para `sigma_baseline` | Média | Alto | Implementar `COALESCE(STDDEV(...), sigma_baseline)` na query; adicionar caso de teste com valores idênticos |
| Trocar σ=0.1 por σ dinâmico altera distribuição Monte Carlo — Suite Big Dig pode regredir silenciosamente para projetos de infraestrutura (σ=0.20) | Média | Alto | Executar Big Dig com σ por setor ANTES de substituir hardcoded; aceitar apenas se RMSE não piora |
