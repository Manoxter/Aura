## User Story
Como PM usando as Funções Prazo e Custo,
quero ver nos gráficos de burndown e Curva S uma faixa sombreada indicando a região de contingência e uma linha vermelha tracejada no teto absoluto,
para que a linguagem visual das zonas seja consistente em todos os painéis do motor.

---

## Acceptance Criteria

### AC-1 — Burndown (Função Prazo): banda de contingência de prazo
- [ ] Faixa amarela entre `x = duracao_caminho_critico` e `x = prazo_total` no eixo X (tempo)
- [ ] Faixa com opacidade 20%, cor amber
- [ ] Linha vertical vermelha tracejada em `x = prazo_total` (teto absoluto do prazo)
- [ ] Tooltip ao hover na faixa: "Zona Amarela — consumindo folga de prazo"
- [ ] Linha vermelha tracejada com label: "Teto de Prazo"

### AC-2 — Curva S (Função Custo): banda de contingência de custo
- [ ] Faixa amarela entre `y = orcamento_operacional` e `y = orcamento_total` no eixo Y (custo)
- [ ] Faixa com opacidade 20%, cor amber
- [ ] Linha horizontal vermelha tracejada em `y = orcamento_total` (teto absoluto de custo)
- [ ] Tooltip ao hover na faixa: "Zona Amarela — consumindo reserva de contingência"
- [ ] Linha vermelha tracejada com label: "Teto de Orçamento"

### AC-3 — Consistência visual com o triângulo
- [ ] Mesmas cores usadas no TrianglePlotter (amber 20%, red 15% / tracejado)
- [ ] Acima do teto: área não renderiza dados (Zona Nula não existe visualmente)

### AC-4 — Dados corretos
- [ ] `orcamento_operacional` calculado com `percentual_contingencia` do TAP (não hardcoded)
- [ ] `duracao_caminho_critico` vem de `caminho_critico_baseline_dias` do projeto
