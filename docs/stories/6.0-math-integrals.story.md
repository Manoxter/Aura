## Contexto
O modelo Aura V1 forçava uma simplificação geométrica onde o projeto nascia sempre como um triângulo equilátero ($C=1,E=1,P=1$), por divisão direta das tangentes base. Isso removia o peso intrínseco (a distorção natural de projetos do tipo "front-loaded"). A transição para a geometria natural exige a substituição desse conceito passivo por Integrais de Acumulação.

## Objetivos (Acceptance Criteria)
- [ ] O eixo $X$ para a área do Matriz deve se tornar uma constante temporal bruta conectada à extensão planejada do Calendário.
- [ ] Criar nova função `calcularIntegralAcumulada(curva, limite)` em `math.ts`.
- [ ] O algoritmo mapeará os pontos críticos (*maximas/minimas) na derivação da Curva de Orçamento/Duração*, extraindo pontos vitais. Em cima deles, aplicar Regressão Linear Múltipla OLS. Esta Reta-Tendência substituirá as hastes originais $C$ e $P$.
- [ ] O baseline $t=0$ NÃO resultará mais obrigatoriamente num triângulo quilátero. Exibirá exatamente a deformação proposta desde o começo do projeto.
- [ ] Criar lógica "Zona de Intersecção Intergral (ZII)". A ZII atuará como o novo nível "Tolerável/Amortecido" de Decisão do Mated.
- [ ] O tipo de triângulo gerado ($Acutângulo$, $Obtusângulo$, etc) será retornado no payload `CDTResult` como String Metadata para ativar painéis de UX de mitigação (ex: Claro protocolo de Obtusângulo -> Clairaut).

## Validação Acadêmica (@qa e @roberta)
- Os Testes Unitários devem garantir que um projeto 80% concentrado nos primeiros 10 dias rompa uma angulação de X graus criando um Triângulo Obtuso crasso.
