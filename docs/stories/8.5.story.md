# Story 8.5 — Fix RLS: Substituir `auth.uid()` por Subquery Otimizada

## User Story
Como sistema de segurança do Aura,
quero que todas as policies RLS usem `(SELECT auth.uid())` em vez de `auth.uid()` diretamente,
para que o Supabase não execute a função para cada linha da tabela (performance e segurança).

## Background
`auth.uid()` chamado diretamente em policies RLS é re-avaliado por linha — comportamento ineficiente
e potencialmente inseguro. A recomendação do Supabase Advisor é usar subquery `(SELECT auth.uid())`
que é avaliada uma vez por statement.

## Riscos
| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| DROP POLICY + CREATE POLICY em tabelas com dados de produção pode criar janela de vulnerabilidade durante a transação — políticas ausentes deixam tabela exposta | Média | Alto | Executar em transação única (`BEGIN ... COMMIT`); validar que DROP e CREATE estão na mesma transação atômica |
| `supabase get_advisors` pode retornar warnings de outras fontes além de RLS — confundir 0 warnings como "0 warnings de RLS" pode mascarar outros problemas | Baixa | Baixo | Filtrar output de `get_advisors` especificamente por warnings de tipo RLS antes de declarar Done |
