## User Story
Como sistema de calibração do Aura,
quero uma Edge Function que atualiza as zonas MATED ao arquivar um projeto,
para que os limiares de risco fiquem progressivamente calibrados com dados empíricos reais.

## Background
Ao arquivar projeto, o evento deve:
1. Inserir linha em `aura_calibration_events` (tipo='empirico')
2. Recalcular média e σ do setor com o novo dado
3. Atualizar `aura_setor_config` se modo empírico ativado (n≥30)
4. Registrar `arquivado_em`, `modo` (normal|invertido), `area_final`, `mated_medio`, `sdo_score`

## Riscos
| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| Edge Function Deno não tem acesso ao mesmo contexto de auth JWT do cliente Next.js — RLS pode bloquear INSERT se Bearer token não for passado corretamente no header | Alta | Alto | Implementar chamada com `Authorization: Bearer ${session.access_token}` no fetch do frontend; testar com Postman antes do deploy |
| `STDDEV()` calculado no body da Edge Function (não em SQL) exige buscar todos os eventos do setor — custo de performance cresce com n | Baixa | Médio | Mover cálculo STDDEV para query SQL inline no Supabase (`rpc()` ou query direta); não buscar registros em TypeScript para calcular |
