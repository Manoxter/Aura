## User Story

Como motor matemático do Aura,
quero classificar automaticamente a morfologia do triângulo de monitoramento após cada recalibração,
para que o sistema ative o protocolo correto (Agudo, β ou γ), detecte estados singulares antes de qualquer fitting,
e exponha os ângulos α, ω, ε com as métricas Prometeu Intrínseco para consumo pela camada de UI (Story 2.0-ui).

---

## Background

A Síntese de Clairaut (SC) é a classificação que roda imediatamente após o CEt, usando a lei dos
cossenos sobre os lados E (Esforço/Custo), P (Prazo), O (Orçamento/Origem).

Ela determina qual protocolo de análise está ativo:
- **Protocolo Agudo**: triângulo saudável, funções normais
- **Protocolo β** (E-O obtuso): orçamento em colapso — função custo refletida
- **Protocolo γ** (E-P obtuso): prazo em colapso — função prazo transladada
- **Estado Singular**: ângulo reto detectado — HALT obrigatório, sem fitting possível

Os três ângulos internos α (Absorção), ω (Entrega), ε (Equilíbrio) formam o **Painel Clairaut**,
com três modos de visualização configuráveis pelo usuário (Triângulo Vivo, Painel de Instrumentos, Radar Angular).

O **Prometeu Intrínseco** (IR, Rα, Rω) é derivado automaticamente dos ângulos — sem nova tabela de banco.

- [x] AC-5: **Protocolo Agudo** — nenhuma alteração nas funções tangentes (comportamento atual mantido)
- [x] AC-6: **Protocolo γ** — função prazo transladada paralelamente até V_origem (vértice mais próximo à origem); `m` preservado integralmente

## Riscos
| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| `Math.acos()` retorna `NaN` para valores fora de `[-1, 1]` devido a arredondamento de ponto flutuante — pré-gate de singularidade pode falhar | Alta | Alto | Clampar entrada: `Math.acos(Math.max(-1, Math.min(1, cosAngulo)))` |
| Invariante visual (clipar projeções) pode requerer refactoring profundo na renderização do triângulo | Média | Alto | Implementar como camada de pós-processamento de coordenadas, independente da lógica de cálculo |
| Três modos de UI geram 3× o trabalho de testes visuais | Média | Médio | @e2e-tester cobre os 3 modos com visual regression screenshots |
| `recharts` pode não suportar radar com escalas invertidas por eixo — modo radar pode precisar de `d3` | Média | Médio | Prototipar modo radar primeiro; se inviável com recharts, usar d3 apenas para este componente |
