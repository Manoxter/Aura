## User Story
Como motor matemático do Aura,
quero uma função `classificarCandidatoCEt()` em `zones.ts` que classifique cada candidato do CEt nas zonas Verde/Amarela/Vermelha/Cinza/Nula com suas consequências em dias e R$,
para que o PM saiba o custo operacional real de cada decisão disponível.

---

## Acceptance Criteria

### AC-1 — Arquivo `zones.ts` criado
- [ ] `src/lib/engine/zones.ts` criado com:
  ```ts
  export type ZonaOperacional = 'verde' | 'amarela' | 'vermelha' | 'cinza' | 'nula'

  export interface ClassificacaoCEt {
    zona: ZonaOperacional
    prazo_real_dias: number       // lado_P desnormalizado em dias
    custo_real_brl: number        // lado_O desnormalizado em R$
    dias_extra: number            // acima do caminho crítico (0 se verde)
    custo_extra_brl: number       // acima do operacional (0 se verde)
    bloqueado: boolean            // true apenas para Zona Nula
    consequencia: string          // texto descritivo para o PM
  }

  export function classificarCandidatoCEt(
    candidato: { P: number; O: number },
    limites: LimitesZona
  ): ClassificacaoCEt
  ```

### AC-2 — Lógica de classificação correta
- [ ] **Verde:** `P ≤ 1.0` E `O ≤ 1.0` (dentro do operacional)
- [ ] **Amarela:** P dentro da folga OU O dentro da contingência (mas não ambos no verde)
  - P: `1.0 < P ≤ prazo_total / caminho_critico`
  - O: `1.0 < O ≤ 1 + percentual_contingencia/100`
- [ ] **Vermelha:** P esgota folga OU O esgota contingência
  - P: `P = prazo_total / caminho_critico` (±0.01 tolerância)
  - O: `O = 1 + percentual_contingencia/100` (±0.01 tolerância)
- [ ] **Cinza:** `P > prazo_total / caminho_critico` (além do prazo total — tempo existe)
- [ ] **Nula:** `O > 1 + percentual_contingencia/100` (além do orçamento total — dinheiro não existe)
- [ ] Zona Nula retorna `bloqueado: true`

### AC-3 — Desnormalização (dias e R$)
- [ ] `prazo_real_dias = candidato.P × caminho_critico_baseline_dias`
- [ ] `custo_real_brl = candidato.O × orcamento_operacional`
- [ ] `dias_extra = max(0, prazo_real_dias - caminho_critico_baseline_dias)`
- [ ] `custo_extra_brl = max(0, custo_real_brl - orcamento_operacional)`

### AC-4 — Renomeação `cet-dimension.ts`
- [ ] `faixa_contingencia` renomeado para `faixa_ajuste` em `cet-dimension.ts`
- [ ] `ZONA_CET_LABELS` e `ZONA_CET_DESCRICAO` atualizados
- [ ] Todos os imports/usos de `faixa_contingencia` atualizados no projeto
- [ ] Nenhum teste quebra pela renomeação

### AC-5 — Testes unitários
- [ ] Teste: candidato com P=0.8, O=0.9 → Verde
- [ ] Teste: candidato com P=1.1 (dentro da folga) → Amarela
- [ ] Teste: candidato com O=1.2 (contingência 15%) → Amarela
- [ ] Teste: candidato com O=1.16 (esgota contingência 15%) → Vermelha
- [ ] Teste: candidato com P além do prazo total → Cinza
- [ ] Teste: candidato com O=1.20 (contingência 15%, orcTotal=100k) → Nula, bloqueado=true
- [ ] Teste: desnormalização — P=1.5, caminho=120 dias → prazo_real=180, dias_extra=60
