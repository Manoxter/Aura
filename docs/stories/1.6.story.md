## User Story
Como motor matemático do Aura,
quero uma função canônica `seedCustosTarefas()` que distribua o orçamento base proporcionalmente à duração de cada tarefa quando `custosTarefas` estiver vazio,
para que a projeção financeira e o CDT funcionem sem setup manual de custos por tarefa.

---

## Acceptance Criteria

### AC-1 — Função seedCustosTarefas() em math.ts
- [ ] `seedCustosTarefas(tarefas, orcamentoBase)` exportada de `math.ts`
- [ ] Retorna `Record<string, number>` com custo proporcional à `duracao_estimada` de cada tarefa
- [ ] Se `orcamentoBase <= 0` ou `tarefas.length === 0`: retorna objeto vazio `{}`
- [ ] Se todas as durações são zero: distribui igualmente (`orcamentoBase / n`)
- [ ] Soma dos valores retornados ≈ `orcamentoBase` (tolerância ±1 centavo por arredondamento)

### AC-2 — Substituição dos fallbacks M7 inline

## Dev Notes
- Distribuição proporcional: `custo_i = orcamentoBase × (duracao_i / duracao_total)`
- `duracao_total` = soma de todas as `duracao_estimada` (não caminho crítico)
- Fallback tarefas com duracao=0: recebem peso 1 (igual às demais com duracao=0)
- A função NÃO persiste no banco — é seed computado em memória para uso no render
- Débito técnico M7 de `technical-debt-assessment.md` coberto por esta story
