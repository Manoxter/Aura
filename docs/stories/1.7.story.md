# Story 1.7 — Projeção CDT (+5 dias) Validada contra CEt
**Épico:** EP-01 Fundamentos Geométricos (Motor Core)
**Sprint:** C-CEt
**Status:** Done
**Agentes:** @dev (Dex) + @aura-math
**Pré-requisito:** Story 1.1 (checkCETDupla), Story 1.2 (NVO)

---

## Objetivo

Quando o motor projeta o CDT para "+5 dias no futuro" (projeção de tendência linear),
o triângulo projetado DEVE ser validado pela CEt Dupla antes de ser exibido ao usuário.
Se a projeção gera um triângulo inválido, ela não é exibida.

## Decisões Técnicas

- **Projeção linear simples:** taxa = `(ladoAtual - 1.0) / diaAtual`. Suficiente para Story 1.7.
  Story 1.3 implementará burndown correto com ES/EF.
- **Nomenclatura CEt:** `violatedSide` usa `'O'` (Orçamento) internamente — mesmo que o campo
  `ProjecaoCDT` exponha `C` como nome do lado. Documentado nos testes.
- **Reuso de `checkCETDupla`:** não reimplementa a lógica de CEt Dupla — delega completamente
  a `crisis.ts` (Story 1.1).
- **UI gating:** painel visível sempre que `cdtAtual` existe; conteúdo alterna entre
  valores projetados (válido) ou mensagem de bloqueio (inválido).
