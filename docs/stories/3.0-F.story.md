# Story 3.0-F — ZonaTimeline + Velocidade de Degradação
**Épico:** EP-03 — Calibração e Zonas do Motor CDT
**Status:** Done
**Agentes:** @dev (Dex), @dataviz (Viz), @roberta

## User Story
Como PM usando o Motor Aura,
quero ver um histórico semanal das zonas em que o projeto esteve e um alerta preditivo quando o ritmo atual leva à deterioração,
para que eu saiba não só onde estou, mas para onde estou indo.

---

## Acceptance Criteria

### AC-1 — Tabela `historico_zonas`
- [ ] Migration `20260321300000_historico_zonas.sql` cria tabela:
  ```sql
  historico_zonas (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    projeto_id UUID REFERENCES projetos(id),
    semana DATE,               -- data da segunda-feira da semana
    zona ZonaOperacional,      -- verde | amarela | vermelha | cinza | nula
    distancia_nvo NUMERIC(8,4),
    criado_em TIMESTAMPTZ DEFAULT now()
  )
  ```
- [ ] RLS: somente tenant do projeto acessa
- [ ] Registro inserido automaticamente ao recalcular TA (1 por semana — não duplicar)

### AC-2 — Componente `ZonaTimeline`
- [ ] Novo `src/components/aura/ZonaTimeline.tsx`
- [ ] Exibe linha do tempo semana a semana com cor da zona de cada semana
- [ ] Tooltip ao hover: semana + zona + distância NVO
- [ ] Mínimo 4 semanas para exibir (ocultar se menos dados)
- [ ] Integrado na página do motor Triângulo Matriz abaixo do triângulo

### AC-3 — Velocidade de Degradação
- [ ] Função `calcularVelocidadeDegradacao(historico: HistoricoZona[]): VelocidadeDegradacao` em `execution.ts`
  ```ts
  interface VelocidadeDegradacao {
    tendencia: 'melhora' | 'estavel' | 'piora'
    dias_ate_zona_amarela: number | null  // null se já amarela ou melhorando
    confianca: number  // 0.0–1.0 (baseado em n° de pontos)
  }
  ```
- [ ] Regressão linear sobre os últimos 5–10 pontos de `distancia_nvo`
- [ ] Janela mínima: 3 pontos para calcular (abaixo de 3: retorna `confianca=0, tendencia='estavel'`)
- [ ] @roberta validou que janela de 5 pontos é suficiente para confiança aceitável

### AC-4 — Alerta Preditivo
- [ ] Se `tendencia = 'piora'` E `confianca >= 0.6`: exibe alerta no header do motor:
  "⚠️ Ritmo atual leva à Zona Amarela em ~X dias"
- [ ] Se `tendencia = 'piora'` E `confianca < 0.6`: alerta com "(dados insuficientes para precisão)"
- [ ] Alerta não exibido se projeto já está em Zona Amarela ou pior

## Riscos
| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| Projetos novos não têm histórico suficiente para regressão | Alta | Baixo | Ocultar alerta até 3+ semanas de dados — não exibir com dados insuficientes |
| Upsert semanal pode gerar duplicatas se `recalcularTA()` for chamado múltiplas vezes na mesma semana | Alta | Baixo | Upsert por `(projeto_id, semana)` com `ON CONFLICT DO UPDATE` |
