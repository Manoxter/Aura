## User Story
Como usuário do Klauss com conversas longas,
quero que o histórico seja truncado de forma inteligente para caber em ~4K tokens,
para que o modelo não rejeite requisições por excesso de contexto e as mensagens mais recentes sejam sempre incluídas.

---

## Acceptance Criteria
- [x] AC-1: `truncateHistory()` mantém as mensagens mais recentes que cabem em ~4K tokens
- [x] AC-2: Limite: MAX_HISTORY_CHARS = 16000 (1 token ≈ 4 chars)
- [x] AC-3: Algoritmo percorre histórico em ordem reversa acumulando chars
- [x] AC-4: Para ao exceder o limite — mensagens mais antigas são descartadas
- [x] AC-5: Histórico truncado aplicado antes de enviar ao Groq
- [x] AC-6: TypeCheck: 0 erros
