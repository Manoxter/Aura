## User Story
Como motor matemático do Aura,
quero uma função `inverterCoordenadas(triangle, resultado)` que aplique a transformação geométrica correta para cada regime obtuso,
para que a UI possa exibir o "triângulo meta invertido" como alvo de recuperação quando o projeto sai do regime agudo.

---

## Background Técnico
- **Regime β** (α > 90° no P_vertex): reflete P_vertex cruzando a linha EO. O "mundo invertido" é o triângulo com P no lado oposto de EO — o projeto precisa reduzir P (prazo) para recruzar para o lado agudo.
- **Regime γ** (ω > 90° no E_vertex): reflete E_vertex cruzando a linha PO. O projeto precisa reduzir E (escopo) para recuperar a acutitude.
- Reflexão usa `projectPointToLine()` de `triangle-logic.ts` (Story 1.9 — já exportada).
