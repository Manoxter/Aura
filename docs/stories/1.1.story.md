## User Story
Como motor matemático do Aura,
quero verificar a Condição de Existência do Triângulo em dois momentos distintos,
para que nenhum triângulo inválido seja calculado ou exibido ao PM.

## Acceptance Criteria
- [x] AC-1: CEt verificada nos valores **brutos** (pré-normalização) — se falhar, cálculo bloqueado imediatamente
- [x] AC-2: CEt verificada nos valores **normalizados** (pós-normalização) — se falhar, cálculo bloqueado
- [x] AC-3: Mensagem de erro clara ao PM indicando qual lado viola a desigualdade triangular
- [x] AC-4: Triângulo NÃO plotado se qualquer verificação falhar
- [x] AC-5: Suite Big Dig continua passando 100% (regressão)
- [x] AC-6: Novo teste unitário: valores que passam na CEt bruta mas falham na normalizada

## Tasks
- [x] 1. Localizar função de cálculo CEt em `src/lib/engine/math.ts`
- [x] 2. Adicionar verificação CEt pré-normalização (valores brutos E, P, O)
- [x] 3. Adicionar verificação CEt pós-normalização (valores adimensionais)
- [x] 4. Retornar erro tipado: `{ valid: false, violatedSide: 'E'|'P'|'O', stage: 'pre'|'post' }`
- [x] 5. Bloquear plotagem do triângulo se `valid === false`
- [ ] 6. Exibir alerta na UI com lado violado e etapa
- [x] 7. Escrever testes unitários para ambas as etapas
- [x] 8. Rodar suite Big Dig — confirmar 100% pass
